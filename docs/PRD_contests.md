## 페이지 개요

**목적:**  
광고주(주최 측)가 등록한 공모전/컨테스트 목록을 보여주고, 클릭 시 상세 페이지로 이동할 수 있는 페이지

**접근 경로:**
- 상단 네비게이션 "컨테스트" 탭 클릭
- URL: `/contests` (또는 `/contest` 등 라우트 1개로 통일)

**주요 사용자:**
- 광고주 회원 (컨테스트를 유치하는 주최 측)
- (추후) 참가 희망자 (공모전 목록 조회·상세 확인)

## 컨테스트 데이터 구조 (참고용)

목록/상세 화면 설계의 기준이 되도록 필드를 정리. DB 스키마는 별도 설계 시 참고.

| 구분 | 필드 | 비고 |
|------|------|------|
| 이미지 | 포스터 이미지 | 1장 |
| 기본 | 타이틀 | |
| 기본 | 내용(본문) | 해시태그·출품방법·필수사항 등 포함 가능 |
| 혜택 | 혜택 | 상금 외 구체적 혜택, DB text |
| 플랫폼 | 게시될 플랫폼 | 유튜브, 인스타그램 등 복수 채널 가능, 없을 수 있음 |
| 조건 | 지원조건 | 컨테스트에 따라 지원조건 제한 없을수도 있고 팔로워 N명 이상 등을 있을수도 있음. 추후 일평균 방문자 수 등 확장 가능 |
| 유형 | 콘텐츠 유형 | 초기에는 '숏폼' 하나 고정 예정 |
| 일정 | 접수일정 | |
| 일정 | 발표일정 | |
| 기타 | 기타 | |
| 공통 | 공통 Q&A | 본문 하단에 포함하거나 별도 섹션으로 게시 (중요도는 본문보다 낮음) |
| 공통 | 공통 법적·운영 안내 | |

**공통 Q&A 예시 (참고):**
- 영상 편집 툴 제한, 개인 SNS 없이 참여 가능 여부, 저작권 음원 사용, 팀 최대 인원, 숏폼 외 형식 제출, 수상 후 영상 활용 등

**공통 법적·운영 안내 예시 (참고):**
- 순수 창작물 요건, 초상권 서면 동의, AI 생성 콘텐츠 출처 명시, 제출 영상 반환 불가·홍보 활용, 제3자 권리 침해 시 참가자 책임, 부정행위 시 수상 취소·상금 환수, 출품 시 위 사항 동의 간주 등

## 컨테스트 목록 페이지 구조

### 1. 네비게이션 바 (공통)
- 로고: "Bling" ✨
- 탭 메뉴: 콘텐츠, **컨테스트** (활성), 인플루언서
- 우측: 로그인, 회원가입 (비로그인 시) / 프로필, 알림 (로그인 시)

### 2. 페이지 헤더
**구성 요소:**
- 페이지 타이틀: "컨테스트"
- 서브타이틀: 공모전 목록 안내 문구 1줄 (예: "진행 중인 공모전을 확인하세요")

### 3. 목록 영역

**레이아웃:** 세로 리스트. 각 컨테스트가 **한 행(카드)** 으로 표시됨.

```
┌─────────────────────────────────────────────────────────────┐
│  [포스터 이미지]   │  타이틀                                 │
│   (좌측 고정 비율)  │  내용 요약 (2~3줄 말줄임)                │
│                    │  접수일정 · 발표일정 · 콘텐츠 유형 등     │
│                    │  [자세히 보기] (선택)                    │
└─────────────────────────────────────────────────────────────┘
```

#### 3.1 목록 아이템 (한 행) 상세

**좌측 - 포스터 이미지:**
- 비율: 4:3 또는 16:9 중 1종 권장 (예: 4:3)
- 데스크톱: 최소 너비 예시 280px, 높이 비율 유지
- 클릭 시 해당 컨테스트 상세 페이지로 이동

**우측 - 컨테스트 내용:**
- **타이틀** (필수): 1줄 또는 2줄 말줄임
- **요약/내용 일부**: 본문 앞부분 또는 별도 요약 필드, 2~3줄 말줄임
- **핵심 정보**: 접수일정, 발표일정, 콘텐츠 유형(예: 숏폼) 등 1~2줄
- (선택) CTA: "자세히 보기" 버튼 또는 텍스트 링크. 없으면 카드 전체 클릭으로 상세 이동

#### 3.2 목록 구성
- 컨테스트 카드가 위에서 아래로 반복
- **검색/필터 없음** (초기에는 개수가 많지 않다고 가정)
- 페이지네이션: 5개 단위 또는 무한 스크롤 등 간단히 적용

#### 3.3 빈 상태
- 문구: "등록된 컨테스트가 없습니다"
- (선택) 일러스트 또는 아이콘

### 4. 인터랙션
- 카드(또는 "자세히 보기") 클릭 → 컨테스트 상세 페이지 이동 (`/contests/[id]` 등)
- 새 탭: Ctrl/Cmd + 클릭 (선택)
